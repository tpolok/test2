Les désavantages d'une PWA

Bien que les avantages soient multiples, les désavantages le sont aussi

Impossible de crypter les données pour le mode offline

En effet, il est impossible de crypter les données pour le mode offline

Il faut donc que les données soient potentiellement visible aux yeux de tous ( la base de donnée étant visible via l'outil de développement de Google Chrome )

L'impossibilité de crypter les données vient du fait qu'il st impossible de décrypter les données crypter sans exposer la clé de décryptement ( rendant tout le processus de crypage inutile

Il faut donc que les données ne soient pas sensibles ou de tout simplement interdire l'accès à la page en question lors du mode hors ligne

La connexion à un compte est donc rendu difficile, puisqu'il est inconcevable de stocker en claire le mot de passe de l'utilisateur

Obligation d'homogénéité

Ce qui est disponible en ligne l'est théoriquemet aussi hors ligne.

Cependant, nous voulons parfois implémenter une feature qui sera disponible uniquement sur le site web et non sur l'application mobile

Cette chose implique donc des traitements supplémentaires si la nécessité de faire cela

Tel que rendre invisible le lien pour y accéder en mode hors ligne ainsi que d'y interdire l'accès si la ersonne essaye de forcer

Un cout d'évolution potentiellement plus élevé

Imaginez le scénario suivant :

Votre PWA est opérationnel, votre client en est content mais vous demande une évolution.

Vous acceptez et commencez le développement.

Le fait de rendre cette evolution totalement responsive et suivre les paradigmes des PWA peut vite devenir un enfer

Si votre client demande par exemple de pouvoir prendre en photo des aliments si le client est sur mobile et uniquement si il est en ligne

Cela vous fait un double traitement puisqu'il faut bien vérifier que la personne est bien sur mobile ( et possède donc une caméra ) ainsi qu'un accès a internet

Si votre site était dissocié de l'application mobile, le traitement aurait éé beaucoup plus simple !

Une non utilisation du store possiblement dérangeante

Ne pas passer par le store peut être un avantage en soit comme un désavantage.

En effet, même si ne pas passer par le store peut être bénéfique économiquement parlant, ne pas l'utiliser peut freiner la découverte de la solution par de nouveaux utilisateurs, ce dernier n'étant pa référencé par une application tierce
